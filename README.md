# M²MVT on DAAD-X — Reproduction

Faithful reproduction of the **M²MVT** architecture from:

> *"Early Anticipation of Driving Maneuvers"* (ECCV 2024)  
> Abdul Wasi, Shankar Gangisetty, Shyam Nandan Rai, C.V. Jawahar

Trained and evaluated on the **DAAD-X** dataset (7-class maneuver anticipation).

## Architecture (Paper-Faithful)

```
Multi-view branch (5 views: front, rear, left, right, driver)
  ┌─────────────────────────────────────────────────────────┐
  │ front  → Patch Embed (X_1) ─┐                           │
  │ rear   → Patch Embed (X_2) ─┤                           │
  │ left   → Patch Embed (X_3) ─┼─ concat → z_mv_n         │
  │ right  → Patch Embed (X_4) ─┤                           │
  │ driver → Patch Embed (X_5) ─┘                           │
  │                                                         │
  │ [e_mv_cls | E_mv_eps (10 tokens) | z_mv_n] = z_mv_0    │
  │                    ↓ shared MViT encoder                │
  │                  e_mv_cls [B, D_final]                  │
  └─────────────────────────────────────────────────────────┘

Ego branch (1 view: aria_gaze)
  ┌────────────────────────────────────────────────────────┐
  │ aria_gaze → Patch Embed (X_ego)                        │
  │ [e_ev_cls | E_ev_eps (2 tokens) | tokens] = z_ev_0     │
  │                    ↓ same shared MViT encoder           │
  │                  e_ev_cls [B, D_final]                  │
  └────────────────────────────────────────────────────────┘

Classification head:
  concat(e_mv_cls, e_ev_cls) → LayerNorm → Dropout → Linear → 7 classes
```

Key paper statements implemented verbatim:
- *"Separate projections are used for all five views."* → `mv_projections` (5 independent `nn.Linear`)
- *"Embeddings from multiple views are fused into a combined sequence."* → `torch.cat(mv_tokens, dim=1)`
- *"A joint CLS token e_mv_cls is learned for this branch."* → `self.e_mv_cls`
- *"For the ego view, a separate CLS token e_ev_cls is learned."* → `self.e_ev_cls`
- *"The classification tokens are concatenated before anticipation."* → `torch.cat([e_mv_cls, e_ev_cls], dim=1)`
- *"For M2MVT, we pass different modalities through the same encoder."* → single `self.encoder` called for both branches
- N=12 episodic memory tokens; 10 for MV, 2 for EV (Table 5b: best performance)

## Data Preparation

### DAAD-X structure expected:
```
/path/to/DAAD-X/
├── front/       ← all front-view mp4 files
├── rear/
├── left/
├── right/
├── driver/
├── aria_gaze/
├── train.csv    ← generated by prepare_splits.py
├── val.csv
└── test.csv
```

### Generate train/val/test splits from master annotation CSV:
```bash
python tools/prepare_splits.py \
    --ann-csv /path/to/DAAD-X/annotations.csv \
    --data-dir /path/to/DAAD-X \
    --train-ratio 0.70 \
    --val-ratio   0.15 \
    --seed 0
```

Annotation CSV format (one row per sample):
```
filename.mp4, maneuver_id, start_frame, [17-dim explanation vector]
```
Only `filename` (col 0) and `maneuver_id` (col 1) are used. The 17-dim vector is ignored.

## Training

### Fresh training:
```bash
python tools/train.py \
    --cfg configs/DAAD/M2MVT_s_16x4.yaml \
    DATA.PATH_TO_DATA_DIR /path/to/DAAD-X \
    OUTPUT_DIR ./output \
    NUM_GPUS 4 \
    TRAIN.BATCH_SIZE 4
```

### Resume from checkpoint:
```bash
python tools/train.py \
    --cfg configs/DAAD/M2MVT_s_16x4.yaml \
    DATA.PATH_TO_DATA_DIR /path/to/DAAD-X \
    OUTPUT_DIR ./output \
    TRAIN.AUTO_RESUME True
```

### Multi-GPU (DataParallel):
```bash
python tools/train.py \
    --cfg configs/DAAD/M2MVT_s_16x4.yaml \
    DATA.PATH_TO_DATA_DIR /path/to/DAAD-X \
    NUM_GPUS 8 \
    TRAIN.BATCH_SIZE 4
```

Paper settings (from implementation details):
- AdamW, LR=1e-4, cosine decay, 30-epoch linear warmup
- 200 epochs total, batch=4 per GPU, 4×RTX 2080
- 16×4 temporal sampling (16 frames, stride 4 → 64-frame span)
- Resolution 224×224

## Inference

### Full M²MVT inference (all 6 views):
```bash
python tools/infer.py \
    --cfg configs/DAAD/M2MVT_s_16x4.yaml \
    --checkpoint output/checkpoints/best.pth \
    --split test \
    DATA.PATH_TO_DATA_DIR /path/to/DAAD-X
```

### Single-view inference (professor-requested ablation):

Any single view can be used alone:
```bash
# Front view only
python tools/infer.py \
    --cfg configs/DAAD/M2MVT_s_16x4.yaml \
    --checkpoint output/checkpoints/best.pth \
    --split test \
    --active-views front \
    DATA.PATH_TO_DATA_DIR /path/to/DAAD-X

# Rear view only
python tools/infer.py ... --active-views rear

# Left view only
python tools/infer.py ... --active-views left

# Right view only
python tools/infer.py ... --active-views right

# Driver (in-cabin) view only
python tools/infer.py ... --active-views driver

# Gaze (ego) view only
python tools/infer.py ... --active-views aria_gaze
```

### Multi-view subsets:
```bash
# Front + Gaze only (out-cabin + ego, no side/rear)
python tools/infer.py ... --active-views front aria_gaze

# Out-cabin only (no gaze)
python tools/infer.py ... --active-views front rear left right driver
```

### Save predictions to JSON:
```bash
python tools/infer.py \
    --cfg configs/DAAD/M2MVT_s_16x4.yaml \
    --checkpoint output/checkpoints/best.pth \
    --split test \
    --active-views front \
    --output-json predictions_front_only.json \
    DATA.PATH_TO_DATA_DIR /path/to/DAAD-X
```

**No retraining required** for single-view inference — the same checkpoint is reused.
Inactive views are zero-filled at inference time to preserve the sequence structure.
